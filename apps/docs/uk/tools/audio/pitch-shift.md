# Зсув висоти тону {#pitch-shift}

Підвищуйте або знижуйте висоту тону аудіофайлу на кілька півтонів без зміни його швидкості відтворення.

## Кінцева точка API {#api-endpoint}

`POST /api/v1/tools/audio/pitch-shift`

Приймає багаточастинні дані форми з аудіофайлом та JSON-полем `settings`.

## Параметри {#parameters}

| Параметр | Тип | Обов'язковий | Типове значення | Опис |
|-----------|------|----------|---------|-------------|
| semitones | integer | Ні | `3` | Півтони для зсуву (-12 до 12). Має бути ненульовим. |

## Приклад запиту {#example-request}

```bash
curl -X POST http://localhost:1349/api/v1/tools/audio/pitch-shift \
  -H "Authorization: Bearer si_your-api-key" \
  -F "file=@audio.mp3" \
  -F 'settings={"semitones": -5}'
```

## Приклад відповіді {#example-response}

```json
{
  "jobId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "downloadUrl": "/api/v1/download/a1b2c3d4-e5f6-7890-abcd-ef1234567890/audio.mp3",
  "originalSize": 4500000,
  "processedSize": 4500000
}
```

## Примітки {#notes}

- Додатні значення підвищують висоту тону; від'ємні знижують її.
- Зсув на 12 півтонів дорівнює одній октаві вгору; -12 дорівнює одній октаві вниз.
- Тривалість відтворення залишається незмінною незалежно від величини зсуву.
- Вихід зазвичай зберігає вхідний контейнер. Вхід AAC записується як M4A, а непідтримувані входи лише для декодування переходять на MP3.
