# Зшивання / Об’єднання {#stitch-combine}

Об’єднайте кілька зображень поруч, з вертикальним накладанням або в сітці. Підтримує вирівнювання, проміжок, межу, радіус кута та кілька режимів зміни розміру.

## Кінцева точка API {#api-endpoint}

`POST /api/v1/tools/image/stitch`

## Параметри {#parameters}

| Параметр | Тип | Обов’язковий | За замовчуванням | Опис |
|-----------|------|----------|---------|-------------|
| direction | string | Ні | `"horizontal"` | Напрямок компонування: `horizontal`, `vertical`, `grid` |
| gridColumns | integer | Ні | 2 | Кількість стовпців, коли напрямок — `grid` (від 2 до 100) |
| resizeMode | string | Ні | `"fit"` | Як зображення змінюють розмір: `fit`, `original`, `stretch`, `crop` |
| alignment | string | Ні | `"center"` | Вирівнювання поперечної осі: `start`, `center`, `end` |
| gap | number | Ні | 0 | Проміжок між зображеннями у пікселях (від 0 до 1000) |
| border | number | Ні | 0 | Ширина зовнішньої межі у пікселях (від 0 до 500) |
| cornerRadius | number | Ні | 0 | Радіус кута, застосований до кінцевого виводу (від 0 до 500) |
| backgroundColor | string | Ні | `"#FFFFFF"` | Колір фону/межі у шістнадцятковому форматі (напр. `#FF0000`) |
| format | string | Ні | `"png"` | Формат виводу: `png`, `jpeg`, `webp`, `avif`, `jxl` |
| quality | number | Ні | 90 | Якість виводу (від 1 до 100) |

## Приклад запиту {#example-request}

```bash
curl -X POST http://localhost:1349/api/v1/tools/image/stitch \
  -F "file=@image1.png" \
  -F "file=@image2.png" \
  -F "file=@image3.png" \
  -F 'settings={"direction":"horizontal","resizeMode":"fit","gap":10,"backgroundColor":"#FFFFFF","format":"png"}'
```

## Примітки {#notes}

- Потрібно щонайменше 2 зображення. Завантажте кілька файлів зображень у багаточастинному запиті.
- Підтримує вхідні формати HEIC, RAW, PSD та SVG (автоматично декодуються).
- Режими зміни розміру:
  - `fit` - Масштабує зображення відповідно до найменшого розміру вздовж осі об’єднання.
  - `original` - Зберігає оригінальні розміри (може призвести до нерівних країв).
  - `stretch` - Примусово приводить зображення до найменшого розміру без збереження співвідношення сторін.
  - `crop` - Обрізає зображення з покриттям, щоб відповідати найменшому розміру.
- У режимі `grid` клітинки набувають розмірів, що відповідають медіанним розмірам усіх зображень.
- `cornerRadius` застосовується до всього кінцевого виводу, а не до окремих зображень.
- Розмір полотна обмежено конфігурацією сервера `MAX_CANVAS_PIXELS`, щоб запобігти вичерпанню пам’яті.
