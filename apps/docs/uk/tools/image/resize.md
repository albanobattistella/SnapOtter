# Зміна розміру {#resize}

Змінюйте розмір зображень, вказуючи точні розміри в пікселях, коефіцієнт масштабу у відсотках або режим вписування, який контролює, як зображення адаптується до цільових розмірів.

## Кінцева точка API {#api-endpoint}

`POST /api/v1/tools/image/resize`

Приймає дані форми multipart із файлом зображення та полем JSON `settings`.

## Параметри {#parameters}

| Параметр | Тип | Обов'язковий | За замовчуванням | Опис |
|-----------|------|----------|---------|-------------|
| width | integer | Ні | - | Цільова ширина в пікселях (максимум 16383) |
| height | integer | Ні | - | Цільова висота в пікселях (максимум 16383) |
| fit | string | Ні | `"contain"` | Як зображення вписується в розміри: `contain`, `cover`, `fill`, `inside`, `outside` |
| withoutEnlargement | boolean | Ні | `false` | Запобігти збільшенню, якщо зображення менше за ціль |
| percentage | number | Ні | - | Масштабувати за відсотком (наприклад, 50 для половинного розміру) |

Потрібно вказати принаймні один з `width`, `height` або `percentage`.

### Режими вписування {#fit-modes}

- **contain** - Змінити розмір, щоб вписатися в розміри, зберігаючи співвідношення сторін (може залишити порожній простір)
- **cover** - Змінити розмір, щоб покрити розміри, зберігаючи співвідношення сторін (може обрізати)
- **fill** - Розтягнути точно до розмірів (ігнорує співвідношення сторін)
- **inside** - Як `contain`, але тільки зменшує, ніколи не збільшує
- **outside** - Як `cover`, але тільки зменшує, ніколи не збільшує

## Приклад запиту {#example-request}

```bash
curl -X POST http://localhost:1349/api/v1/tools/image/resize \
  -H "Authorization: Bearer si_your-api-key" \
  -F "file=@photo.jpg" \
  -F 'settings={"width": 800, "height": 600, "fit": "contain"}'
```

Зміна розміру за відсотком:

```bash
curl -X POST http://localhost:1349/api/v1/tools/image/resize \
  -H "Authorization: Bearer si_your-api-key" \
  -F "file=@photo.jpg" \
  -F 'settings={"percentage": 50}'
```

## Примітки {#notes}

- Максимальний розмір становить 16383 пікселі по будь-якій осі (обмеження Sharp/libvips).
- Вихідний формат збігається з вхідним. Вхідні дані HEIC, RAW, PSD та SVG автоматично декодуються перед обробкою.
- Орієнтація EXIF автоматично застосовується перед зміною розміру.
- Прапорець `withoutEnlargement` корисний для пакетної обробки, де деякі зображення можуть вже бути меншими за ціль.
