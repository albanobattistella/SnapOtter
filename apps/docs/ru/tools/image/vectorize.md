# Изображение в SVG {#image-to-svg}

Векторизуйте растровые изображения в SVG с помощью алгоритмов трассировки. Поддерживает чёрно-белую трассировку (potrace) и полноцветную многослойную векторизацию.

## Конечная точка API {#api-endpoint}

`POST /api/v1/tools/image/vectorize`

## Параметры {#parameters}

| Параметр | Тип | Обязательный | По умолчанию | Описание |
|-----------|------|----------|---------|-------------|
| colorMode | string | Нет | `"bw"` | Режим трассировки: `bw` (чёрно-белый) или `color` (многоцветные слои) |
| threshold | number | Нет | 128 | Порог яркости для Ч/Б-режима (от 0 до 255). Пиксели ниже становятся чёрными. |
| colorPrecision | number | Нет | 6 | Точность квантования цвета для цветного режима (от 1 до 16). Более высокие значения дают больше различимых цветовых слоёв. |
| layerDifference | number | Нет | 6 | Минимальная разница цвета между слоями в цветном режиме (от 1 до 128) |
| filterSpeckle | number | Нет | 4 | Минимальная площадь для трассируемых фигур в пикселях (от 1 до 256). Удаляет шум/крапинки. |
| pathMode | string | Нет | `"spline"` | Сглаживание контура: `none` (зубчатый), `polygon` (прямые сегменты), `spline` (плавные кривые) |
| cornerThreshold | number | Нет | 60 | Пороговый угол для обнаружения углов в цветном режиме (от 0 до 180 градусов) |
| invert | boolean | Нет | `false` | Инвертировать изображение перед трассировкой (поменять чёрное/белое) |

## Пример запроса {#example-request}

```bash
curl -X POST http://localhost:1349/api/v1/tools/image/vectorize \
  -F "file=@logo.png" \
  -F 'settings={"colorMode":"bw","threshold":128,"filterSpeckle":4,"pathMode":"spline"}'
```

### Цветная векторизация {#color-vectorization}

```bash
curl -X POST http://localhost:1349/api/v1/tools/image/vectorize \
  -F "file=@illustration.png" \
  -F 'settings={"colorMode":"color","colorPrecision":8,"layerDifference":6,"filterSpeckle":4}'
```

## Примечания {#notes}

- Вывод всегда представляет собой файл SVG независимо от формата ввода.
- Поддерживает входные форматы HEIC, RAW, PSD и SVG (автоматически декодируются в растр перед трассировкой).
- Ч/Б-режим использует алгоритм potrace. Изображение сначала преобразуется в оттенки серого, затем пороговое значение приводит его к чисто чёрному/белому перед трассировкой.
- Цветной режим использует многослойный подход: изображение квантуется в цветовые слои, каждый из которых трассируется отдельно и накладывается стопкой в выводе SVG.
- Более низкие значения `filterSpeckle` сохраняют больше деталей, но дают более крупные файлы SVG с большим количеством контуров.
- Настройка `pathMode` значительно влияет на размер файла: `none` даёт больше всего контуров, `spline` даёт самый плавный (и обычно самый маленький) вывод.
- Для наилучших результатов с логотипами и иконками используйте Ч/Б-режим с чистым высококонтрастным вводом. Для фотографий или иллюстраций используйте цветной режим с более высоким `colorPrecision`.
