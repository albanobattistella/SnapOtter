# Гистограмма {#histogram}

Создаёт RGB-гистограмму из изображения. Возвращает PNG-изображение гистограммы вместе со статистикой по каналам и необработанными данными гистограммы из 256 корзин в JSON-ответе.

## Конечная точка API {#api-endpoint}

`POST /api/v1/tools/image/histogram`

Принимает данные формы multipart с файлом изображения и полем JSON `settings`.

## Параметры {#parameters}

| Параметр | Тип | Обязательный | По умолчанию | Описание |
|-----------|------|----------|---------|-------------|
| scale | string | Нет | `"linear"` | Масштаб оси Y: `linear` или `log` |

## Пример запроса {#example-request}

```bash
curl -X POST http://localhost:1349/api/v1/tools/image/histogram \
  -H "Authorization: Bearer si_your-api-key" \
  -F "file=@photo.jpg" \
  -F 'settings={"scale": "linear"}'
```

## Примечания {#notes}

- Поле `downloadUrl` указывает на отрендеренную PNG-гистограмму, показывающую распределения каналов R, G, B и яркости.
- `bins` содержит необработанные массивы из 256 значений для каждого канала (красный, зелёный, синий, яркость), пригодные для построения собственных визуализаций.
- `stats` предоставляет среднее, медиану и стандартное отклонение по каждому каналу.
- `mean` и `max` являются обратно совместимыми сокращёнными полями.
- Используйте масштаб `log`, когда в гистограмме доминируют несколько пиков, а вы хотите увидеть детали в нижних корзинах.
- Входные данные HEIC, RAW, PSD и SVG декодируются автоматически перед анализом.
