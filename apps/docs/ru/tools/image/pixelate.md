# Пикселизация {#pixelate}

Применение эффекта пикселизации ко всему изображению или к определённой прямоугольной области. Полезно для сокрытия чувствительного содержимого, такого как лица, номерные знаки или личная информация.

## Эндпоинт API {#api-endpoint}

`POST /api/v1/tools/image/pixelate`

Принимает multipart form data с файлом изображения и JSON-полем `settings`.

## Параметры {#parameters}

| Параметр | Тип | Обязательный | По умолчанию | Описание |
|-----------|------|----------|---------|-------------|
| blockSize | integer | Нет | `12` | Размер блока пикселей (2-128); большие значения дают более грубую пикселизацию |
| region | object | Нет | - | Ограничить пикселизацию прямоугольником (см. ниже) |

### Объект области {#region-object}

| Поле | Тип | Обязательный | Описание |
|-------|------|----------|-------------|
| left | integer | Да | Отступ слева в пикселях (>= 0) |
| top | integer | Да | Отступ сверху в пикселях (>= 0) |
| width | integer | Да | Ширина области в пикселях (>= 1) |
| height | integer | Да | Высота области в пикселях (>= 1) |

## Пример запроса {#example-request}

Пикселизация всего изображения:

```bash
curl -X POST http://localhost:1349/api/v1/tools/image/pixelate \
  -H "Authorization: Bearer si_your-api-key" \
  -F "file=@photo.jpg" \
  -F 'settings={"blockSize": 20}'
```

Пикселизация определённой области:

```bash
curl -X POST http://localhost:1349/api/v1/tools/image/pixelate \
  -H "Authorization: Bearer si_your-api-key" \
  -F "file=@photo.jpg" \
  -F 'settings={"blockSize": 16, "region": {"left": 100, "top": 50, "width": 200, "height": 150}}'
```

## Пример ответа {#example-response}

```json
{
  "jobId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "downloadUrl": "/api/v1/download/a1b2c3d4-e5f6-7890-abcd-ef1234567890/photo.jpg",
  "originalSize": 2450000,
  "processedSize": 2380000
}
```

## Примечания {#notes}

- Когда `region` не указан, пикселизируется всё изображение.
- Координаты области указываются в пикселях относительно левого верхнего угла изображения. Область должна находиться в пределах границ изображения.
- Выходной формат совпадает с входным. Входные данные в форматах HEIC, RAW, PSD и SVG автоматически декодируются перед обработкой.
